## 2. Bug dates (découvert pendant les tests)

## 4. Store véhicules partagé (enregistrement + synchro toutes fenêtres)

**Problème** : chaque BrowserWindow = processus séparé avec sa copie de `mockVehicules` ; `handleEnregistrer` ne stockait rien → nouvel enregistrement invisible dans Liste/Dashboard.

**Solution** : `src/renderer/src/mock/vehiculesStore.ts`
- Ajouts persistés dans `localStorage` (`tcit_vehicules_added`) — partagé entre toutes les fenêtres Electron
- Hook `useVehicules()` : re-rend via event `storage` (cross-fenêtres) + CustomEvent `tcit:vehicules-changed` (fenêtre émettrice)
- Helpers : `addVehicule`, `nextRef` (max+1, 6 chiffres), `nextId`, `countAddedForDest` (incrément N° IMMAT par destination)
- `handleEnregistrer` ajoute réellement le véhicule (parc = maison de transit, cf. STCA II réel)
- **8 consommateurs convertis** : DashboardHome, ListePage, RechercheWindow, DestinationPage, AssuranceWindows, ListeParDestPage, AnalysePage, MainScreen (statusbar)
- Bonus : récursion infinie latente corrigée dans `closeWindow` (AssuranceWindows)

**Test réel validé** : enregistrement `TEST CLAUDE SYNC` / Voiture / AFO / TOYOTA TEST → Réf 010052, IMMAT C7389 → Dashboard 52→53, jour 2→3, FCFA 520k→530k, barre AFO 6→7, table + Liste + statusbar synchronisées **sans reload**. Store nettoyé après test.
