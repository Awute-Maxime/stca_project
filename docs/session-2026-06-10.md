### 3. Formulaire d'enregistrement — page unique compacte (commit aeed607)
**Problème :** L'utilisateur a montré une capture de l'original STCA-II prouvant que le formulaire tient en UNE page sans scroll.

- Refonte complète de `EnregistrementPage.tsx` avec layout grille compacte
- Suppression des `<Card>` Ant Design (trop de padding)

- Modals de recherche pour Marque/Modèle et Parc/Zone
- `MdiWindow body padding` : 16px → 8px
- `WINDOW_REGISTRY enregistrement height` : 600 → 590px
- **Résultat :** tout visible sans scroll, boutons toujours accessibles

### 4. Architecture MDI multi-BrowserWindow (commit b765f8e) — MAJEUR
**Problème :** Les fenêtres MDI (HTML + `position:fixed`) ne pouvaient pas sortir de la fenêtre Electron hôte. L'utilisateur voulait le comportement STCA-II original où les fenêtres flottent librement sur le bureau.

**Cause racine :** Les éléments HTML ne peuvent PHYSIQUEMENT jamais dépasser le bord de leur fenêtre OS, même avec `position:fixed`.

**Solution :** Chaque fenêtre MDI = une vraie `BrowserWindow` Electron séparée.

**Fichiers créés/modifiés :**
- `src/main/index.ts` — `setupMdiIPC()` : handlers `mdi:open`, `mdi:self:close/minimize/maximize`
- `src/preload/index.ts` — expose `mdiOpen`, `mdiSelfClose/Minimize/Maximize`
- `src/renderer/src/api/electron.ts` — wrappers TypeScript
- `src/renderer/src/pages/MdiWindowHost.tsx` — NEW : page chargée dans chaque BrowserWindow enfant
  - Barre de titre draggable (`WebkitAppRegion: 'drag'`) → drag natif OS
  - Route : `#/mdi/:id`
- `src/renderer/src/App.tsx` — nouvelle route `/mdi/:id`
- `src/renderer/src/windows/MainScreen.tsx` — suppression HTML MdiWindow, appel `electronApi.mdiOpen()`
